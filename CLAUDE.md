## Project Overview

tribucket is a cross-platform package repository providing multiple installation formats from a single source of truth:
- **Homebrew** formulas (`Formula/*.rb`) — macOS/Linux
- **Scoop** manifests (`bucket/*.json`) — Windows
- **Shell scripts** (`scripts/install.sh`, `scripts/install.ps1`) — any platform fallback
- **Portable packages** (`portable/<name>/`) — v2 self-contained portable apps (auto-generated)

v2 adds a **lightweight portable package manager CLI** (`tribucket`) that tracks, checks, and updates installed portable software.

## Critical Constraint

`Formula/*.rb`, `bucket/*.json`, and `portable/<name>/` are **auto-generated** — never edit them directly. All package changes go into `packages/*.json`, then run the generator.

## Commands

### v1 — Generator

```bash
# Generate all Formula + Bucket from packages/*.json
python3 scripts/generate.py

# Generate for a single package
python3 scripts/generate.py --only <name>

# Preview output without writing files
python3 scripts/generate.py --dry-run

# Fast iteration on templates (skip SHA256 computation)
python3 scripts/generate.py --skip-hash

# Validate asset patterns against latest GitHub releases
python3 scripts/generate.py --check-assets

# Generate portable package templates (v2)
python3 scripts/generate.py --portable
python3 scripts/generate.py --only go-wxpush --portable
```

### v2 — CLI

```bash
# Run the tribucket CLI
python3 bin/tribucket --help
python3 bin/tribucket --version

# Install a package
python3 bin/tribucket install <name> [--dir <path>] [--link] [--force]

# Update / check / list
python3 bin/tribucket update <name>
python3 bin/tribucket check <name>
python3 bin/tribucket list

# Track / untrack existing packages
python3 bin/tribucket track <name> <path>
python3 bin/tribucket untrack <name>

# Config
python3 bin/tribucket config list
python3 bin/tribucket config set default_install_dir ~/apps
```

### Testing

```bash
# Run all tests (133 passing)
python3 -m pytest tests/ -v

# Run specific test file
python3 -m pytest tests/test_generate.py -v
```

## Architecture

### Data Flow (v1)

```
packages/*.json  ──→  scripts/generate.py  ──→  Formula/*.rb   (Homebrew)
   (source of truth)                           bucket/*.json  (Scoop)
```

### Data Flow (v2)

```
packages/*.json  ──→  scripts/generate.py  ──→  Formula/*.rb   (Homebrew)
   (source of truth)                           bucket/*.json  (Scoop)
                                              portable/<name>/  (portable templates)
```

`packages/*.json` is the **single source of truth**. `generate.py` produces all three output formats.

### v2 Engine — `lib/tribucket/`

```
lib/tribucket/
├── __init__.py          # Version
├── __main__.py          # python -m tribucket entry
├── cli.py               # argparse CLI (9 commands)
├── config.py            # Path constants, atomic JSON read/write
├── utils.py             # HTTP, SHA256, platform detection, extract_archive
├── track.py             # config.json management (track/untrack/list)
├── check.py             # Version detection (cli→config→fallback chain)
├── mirror.py            # Multi-provider mirror with TTL cache
├── update.py            # Safe update (temp dir relay, backup, file lock)
└── install.py           # First-time install (fetch metadata, download, track)
```

**Key design decisions:**
- Python core engine, Shell only for thin wrappers
- `install.sh` in portable packages delegates to tribucket CLI when available
- Safe updates: download to temp dir → SHA256 verify → replace → backup
- Config stored in `~/.tribucket/config.json` (JSON, key = `owner/repo`)
- Mirror: multi-provider with TTL cache + fallback to direct

### Key Scripts

- `scripts/generate.py` — main generator (templates, hashing, API calls, portable output)
- `scripts/checkver.py` — version detection for custom download sources
- `scripts/add_packages.py` — batch tool to add new packages from GitHub releases
- `scripts/bootstrap.sh` — CLI bootstrap installer (curl | bash)

### Package Modes

**GitHub Release** — uses `repo` + `asset_pattern` to match release assets.

**Custom Download URL** — uses `download_url` + `version` fields for non-GitHub sources.

**Portable Package** — `portable/<name>/` contains `tribucket.json`, `install.sh`, `cmd/tribucket-update.bat`. Generated by `generate.py --portable`.

Platform keys: `linux_amd64`, `linux_arm64`, `darwin_amd64`, `darwin_arm64`, `windows_amd64`, `windows_arm64`. Value `"NO_MATCH"` means unsupported.

### Testing Patterns

Tests use `pytest` with `monkeypatch` to mock HTTP/URL calls and `tmp_path` fixtures for temp files.

- `tests/test_generate.py` — generator + portable output (67 tests)
- `tests/test_tribucket.py` — engine modules (25 tests)
- `tests/test_integration.py` — end-to-end flows (18 tests)
- `tests/test_checkver.py` — version detection (23 tests)

## Adding a Package

1. Create `packages/<name>.json` with required fields
2. Run `python3 scripts/generate.py --only <name>`
3. Run `python3 -m pytest tests/ -v`
4. Commit with conventional commit format (feat/fix/chore/docs/test/ci)

Required fields for GitHub Release: `name`, `repo`, `description`, `binary`, `license`, `homepage`, `asset_pattern`.

For custom sources, add `download_url` (per-platform) and `version`; optionally `checkver` and `autoupdate`.
